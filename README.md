##### 第3章-递归

- 递归只是让解决方案更清晰，并没有性能上的优势。“如果使用循环性能可能更高；如果使用递归，程序可能更容易理解，如果选择要看什么对你来说更重要”


##### 第4章- 快速排序
- 分而治之（divide and conquer，D&C）
- D&C的工作原理：
  1、找出简单的基线条件；（基线条件通常是数组为空或者只包含一个元素）
  2、确定如何缩小问题的规模，使其慢慢靠近基线条件，最终符合基线条件
